# 坐姿提醒器

调用摄像头低频采样（默认每 10 秒分析 1 帧），检测**脖子前倾**、**驼背**、**久坐**，
通过系统弹窗 + 提示音提醒。针对侧面视角设计（笔记本放在外接显示器旁边即可），
全部本地运行，画面不落盘、不上传。支持 macOS 和 Windows。

## 使用步骤

```bash
cd ~/works/bodyjk
source .venv/bin/activate

# 1. 预览：调整摄像头，让画面拍到你的侧面头肩（最好连髋部也拍到），按 q 退出
python main.py --preview

# 2. 校准：坐正（收下巴、耳朵对齐肩膀、背挺直），程序采集 10 秒记录你的基线
python main.py --calibrate

# 3. 开始监控（Ctrl+C 停止）
python main.py

# 想边监控边看画面和角度（试用/调参时），加 --show，按 q 退出
python main.py --show
```

`--show` 模式下画面持续刷新（CPU 占用会高一些），但姿态检测仍按采样间隔执行，
叠加显示的角度、在座时间和下次检测倒计时；提醒逻辑和静默模式完全一样。
日常挂着用不加 `--show`，保持低功耗。

第一次运行时 macOS 会弹出摄像头权限请求，允许即可。

## 日常启动（二选一，均可拖到桌面双击）

- **`菜单栏坐姿监控.command`（推荐）**：常驻屏幕右上角菜单栏，无终端窗口。
  图标即状态：🟢 坐姿 OK（带在座分钟数）、🔴 坏姿势、⚪ 画面没人、⏸ 已暂停、
  🎯 校准中、⚠️ 出错。点开菜单可以看当前角度/在座时长、暂停（开会时用）、
  重新校准（换了摄像头位置后用，坐正后点它等 10 秒即可，不用开终端）、退出。
  日志写到 `menubar.log`。
- **`坐姿监控.command`**：打开一个终端窗口运行监控，日志滚动显示；
  Ctrl+C 或关窗口即停止。

两个脚本里项目路径都是写死的绝对路径，把项目目录挪走后需要同步改 `PROJECT_DIR`。

## Windows 使用

1. 安装 [Python 3.11 或 3.12](https://www.python.org/downloads/)（MediaPipe 暂不支持 3.13）
2. 把项目目录拷到 Windows 机器上（建议 `%USERPROFILE%\works\bodyjk`，
   启动脚本找不到自身目录时会回退到这个路径），然后在项目目录执行：
   ```bat
   py -3.11 -m venv .venv
   .venv\Scripts\pip install -r requirements.txt
   ```
3. 校准和 CLI 用法与 macOS 相同（`.venv\Scripts\python main.py --calibrate` 等）
4. 日常启动：
   - **`托盘坐姿监控.bat`** — 常驻任务栏右下角托盘，图标颜色即状态
     （绿=OK、红=坏姿势、灰=没人、黄=暂停、紫=校准中、橙=出错），
     悬停看角度/在座时长，右键菜单可暂停、重新校准、退出
   - **`坐姿监控.bat`** — 终端窗口版，日志滚动显示
5. 弹窗用系统 MessageBox（置顶、超时自动关闭），声音用系统提示音，无额外依赖；
   摄像头权限在 设置 → 隐私和安全性 → 相机 里允许桌面应用访问

想把 `.bat` 复制到桌面用的话，先把文件里的回退路径改成项目实际路径
（在项目里创建"桌面快捷方式"则不用改）。`requirements.txt` 按平台自动选装：
rumps 只在 macOS 装，pystray/Pillow 只在 Windows 装。

## 代码结构

- `config.py` — 配置默认值和读写
- `posture.py` — MediaPipe 姿态检测 + 角度计算
- `monitor_core.py` — 采样/判定/提醒状态机（CLI 和菜单栏共用）
- `alerts.py` — 弹窗、声音、通知横幅（按平台分支：macOS 用 osascript/afplay，
  Windows 用 MessageBox/winsound）
- `main.py` — CLI 入口（校准/预览/监控），跨平台
- `menubar.py` — macOS 菜单栏常驻应用（rumps）
- `tray_win.py` — Windows 系统托盘常驻应用（pystray）

## 判定逻辑

| 项目 | 依据 | 默认阈值 |
|------|------|----------|
| 脖子前倾 | 肩→耳连线偏离竖直的角度，超过校准基线 | +12° |
| 驼背 | 髋→肩连线偏离竖直的角度，超过校准基线（髋部被桌子挡住则跳过） | +10° |
| 久坐 | 画面里持续有人的时长（短暂离开 <3 分钟不重置） | 50 分钟 |

坏姿势要**连续 3 次采样**（约 30 秒）都超标才提醒，提醒后有 3 分钟冷却期，
避免喝口水低个头就被轰炸。

## 斜视角与选侧

摄像头不在正侧面（比如放在左前方/右前方）时，MediaPipe 两侧关键点可见度接近，
按可见度自动选侧会左右横跳导致角度跳变。因此：

- 校准时会记录**主导侧**并保存到基线里，之后监控锁定这一侧，不再每帧重选
- 如果自动记录的侧不对，在 `config.json` 里手动指定 `track_side`: `"left"` 或 `"right"`
  （按你身体的左右，不是画面的左右；摄像头在你左前方一般用 `"left"`）
- 每次采样连拍 3 帧取中位数（`frames_per_sample`），进一步抑制单帧抖动

## 调参

所有参数在 `config.json` 里，改完重启生效。常用的：

- `sample_interval_sec`：采样间隔，越大越省电（10~20 都合理）
- `neck_delta_deg` / `torso_delta_deg`：误报多就调大，漏报多就调小
- `track_side`：锁定跟踪侧，`"auto"` = 用校准记录的主导侧
- `camera_index`：接了外置摄像头后如果识别错了摄像头，改成 1
- `sit_limit_min` / `sit_realert_min`：久坐上限 / 继续坐着的重复提醒间隔

换了摄像头位置或角度后，**需要重新校准**（`--calibrate`）。

## 性能

- 每 10 秒只做一次姿态推理（MediaPipe Pose，CPU 单帧约几十毫秒），平均 CPU 占用 <1%
- 摄像头保持开启（指示灯常亮）但不持续处理画面；这是为了取帧即时，功耗远低于视频会议
