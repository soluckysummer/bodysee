# 坐姿提醒器 · bodysee

用一颗普通摄像头守护你的颈椎：低频采样（默认每 10 秒分析 1 帧），检测**脖子前倾**、
**驼背**、**久坐**，通过系统弹窗 + 提示音提醒你调整坐姿、起身活动。
支持 macOS 和 Windows，**全部本地运行，画面不落盘、不上传，无任何联网请求**
（仅首次运行下载一次姿态模型）。

附带多款用同一套姿态识别技术制作的体感游戏：挥手切瓜的**体感切水果**、
全身节奏游戏**星核指挥家**，以及用倾身、举手和下蹲控制的三轨跑酷**霓虹疾跑**。

## 实现原理

核心是 Google 的 [MediaPipe](https://ai.google.dev/edge/mediapipe/solutions/vision/pose_landmarker)
**Pose Landmarker（lite 模型，约 5.5MB）**：输入一帧画面，输出人体 33 个关键点
（耳、肩、髋、腕等）的坐标，纯 CPU 推理单帧只要几十毫秒，无需显卡。

**坐姿监视器**在此之上做了三件事：

1. **角度计算**（`posture.py`）：取侧面视角下的关键点，算两个角度——
   肩→耳连线偏离竖直的角度（反映脖子前倾），髋→肩连线偏离竖直的角度（反映驼背）。
2. **个人基线校准**：每个人体型、摄像头摆位都不同，没有放之四海皆准的"标准角度"。
   所以先让你坐正 10 秒，记录你自己的基线角度，之后只看"比基线多偏了几度"。
3. **采样状态机**（`monitor_core.py`）：每 10 秒醒来抓 3 帧取中位数（抑制单帧抖动），
   连续 3 次采样都超标才提醒（约 30 秒，避免低头喝口水就被轰炸），提醒后有冷却期；
   同时累计"画面里持续有人"的时长做久坐提醒。

**切水果游戏**复用同一个模型，换成视频模式逐帧追踪：后台线程 30Hz 读取手腕/食指
关键点，主循环 60fps 渲染；两帧之间手部移动速度超过阈值就产生"手刀"，
与西瓜碰撞判定得分。画面全部程序绘制、音效全部程序合成，仓库里没有一张图片素材。

### 技术栈

| 组件 | 用途 |
|------|------|
| Python 3.11 / 3.12 | 主语言 |
| MediaPipe Tasks (Pose Landmarker lite) | 人体姿态关键点检测 |
| OpenCV (opencv-python) | 摄像头取帧 |
| NumPy | 角度计算、游戏音效合成 |
| pygame | 切水果游戏的渲染和音频 |
| rumps（仅 macOS） | 菜单栏常驻应用 |
| pystray + Pillow（仅 Windows） | 系统托盘常驻应用 |

弹窗和声音直接用系统能力（macOS：osascript / afplay；Windows：MessageBox / winsound），
不引入额外依赖。

## 环境要求（前提条件）

- **操作系统**：macOS 或 Windows 10/11
- **Python 3.11 或 3.12**（[官网下载](https://www.python.org/downloads/)；
  ⚠️ MediaPipe 暂不支持 3.13，版本别装太新）
- **Git**（[下载](https://git-scm.com/downloads)；也可以直接在 GitHub 页面
  Code → Download ZIP，不装 Git）
- **一颗摄像头**：笔记本内置或 USB 外接均可
- 首次运行需要联网一次，自动下载姿态模型（约 5.5MB），之后完全离线

## 安装

### macOS

```bash
# 1. 下载项目
git clone https://github.com/soluckysummer/bodysee.git
cd bodysee

# 2. 创建虚拟环境并安装依赖
python3.11 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

### Windows

```bat
git clone https://github.com/soluckysummer/bodysee.git
cd bodysee

py -3.11 -m venv .venv
.venv\Scripts\pip install -r requirements.txt
```

`requirements.txt` 按平台自动选装：rumps 只在 macOS 装，pystray/Pillow 只在 Windows 装。

首次运行时系统会弹出摄像头权限请求，允许即可
（Windows 在 设置 → 隐私和安全性 → 相机 里允许桌面应用访问）。

## 坐姿监视器：使用步骤

```bash
# macOS 先激活虚拟环境；Windows 把 python 换成 .venv\Scripts\python
source .venv/bin/activate

# 1. 预览：调整摄像头，让画面拍到你的侧面头肩（最好连髋部也拍到），按 q 退出
python main.py --preview

# 2. 校准：坐正（收下巴、耳朵对齐肩膀、背挺直），程序采集 10 秒记录你的基线
python main.py --calibrate

# 3. 开始监控（Ctrl+C 停止）
python main.py

# 想边监控边看画面和角度（试用/调参时），加 --show，按 q 退出
python main.py --show
```

摄像头**放在侧面**效果最好（比如笔记本放在外接显示器旁边，侧对着你）。
`--show` 模式画面持续刷新（CPU 占用高一些），日常挂着用不加它，保持低功耗。
换了摄像头位置或角度后，**需要重新校准**。

### 日常启动（双击即用）

macOS（二选一）：

- **`菜单栏坐姿监控.command`（推荐）**：常驻屏幕右上角菜单栏，无终端窗口。
  图标即状态：🟢 坐姿 OK（带在座分钟数）、🔴 坏姿势、⚪ 画面没人、⏸ 已暂停、
  🎯 校准中、⚠️ 出错。点开菜单可以看当前角度/在座时长、暂停（开会时用）、
  重新校准（坐正后点它等 10 秒即可，不用开终端）、退出。日志写到 `menubar.log`。
- **`坐姿监控.command`**：打开一个终端窗口运行监控，日志滚动显示；
  Ctrl+C 或关窗口即停止。

Windows（二选一）：

- **`托盘坐姿监控.bat`** — 常驻任务栏右下角托盘，图标颜色即状态
  （绿=OK、红=坏姿势、灰=没人、黄=暂停、紫=校准中、橙=出错），
  悬停看角度/在座时长，右键菜单可暂停、重新校准、退出
- **`坐姿监控.bat`** — 终端窗口版，日志滚动显示

> **启动脚本会自动使用自己所在的目录作为项目目录**，clone 到哪里都能直接双击。
> 想放到桌面：macOS 右键脚本 →"制作替身"，Windows 右键 → 发送到 → 桌面快捷方式
> （用替身/快捷方式，别直接复制文件）。如果确实要把脚本文件挪到项目外，
> 打开脚本把顶部的 `PROJECT_DIR`（.bat 里是 `PROJECT_DIR_OVERRIDE`）填成你的项目路径即可。

## 判定逻辑

| 项目 | 依据 | 默认阈值 |
|------|------|----------|
| 脖子前倾 | 肩→耳连线偏离竖直的角度，超过校准基线 | +12° |
| 驼背 | 髋→肩连线偏离竖直的角度，超过校准基线（髋部被桌子挡住则跳过） | +10° |
| 久坐 | 画面里持续有人的时长（短暂离开 <3 分钟不重置） | 50 分钟 |

坏姿势要**连续 3 次采样**（约 30 秒）都超标才提醒，提醒后有 3 分钟冷却期。

### 斜视角与选侧

摄像头不在正侧面（比如放在左前方/右前方）时，MediaPipe 两侧关键点可见度接近，
按可见度自动选侧会左右横跳导致角度跳变。因此：

- 校准时会记录**主导侧**并保存到基线里，之后监控锁定这一侧，不再每帧重选
- 如果自动记录的侧不对，在 `config.json` 里手动指定 `track_side`
  （按你身体的左右，不是画面的左右；摄像头在你左前方一般用 `"left"`）

## config.json 参数详解

`config.json` 在**首次运行时自动生成**在项目目录下（校准结果也写在里面），
改完参数**重启监控生效**。各参数作用：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `camera_index` | `0` | 摄像头编号。接了外置摄像头后如果打开的不是想要的那颗，改成 `1` |
| `track_side` | `"auto"` | 跟踪身体哪一侧：`"left"` / `"right"` / `"auto"`（auto = 用校准时记录的主导侧）。斜视角下选侧不对时手动指定 |
| `frames_per_sample` | `3` | 每次采样连拍几帧取中位数，抑制关键点抖动 |
| `sample_interval_sec` | `10` | 采样间隔（秒），越大越省电，10~20 都合理 |
| `bad_streak_to_alert` | `3` | 连续多少次采样都是坏姿势才提醒（3 次 × 10 秒 = 30 秒） |
| `alert_cooldown_sec` | `180` | 姿势提醒后的冷却时间（秒），避免连环轰炸 |
| `neck_delta_deg` | `12` | 颈部角度超过基线多少度算前倾。**误报多就调大，漏报多就调小** |
| `torso_delta_deg` | `10` | 躯干角度超过基线多少度算驼背。同上 |
| `sit_limit_min` | `50` | 连续在座多少分钟提醒起身 |
| `sit_realert_min` | `10` | 久坐提醒后若继续坐着，每隔多少分钟再提醒 |
| `absence_reset_min` | `3` | 离开画面超过多少分钟算"起身过了"，重置久坐计时 |
| `baseline` | `null` | 校准结果（颈/躯干基线角度 + 主导侧），由 `--calibrate` 自动写入，**不要手改** |

## 附赠：体感切水果小游戏 🍉

久坐提醒响了却懒得动？站起来切两局西瓜。摄像头追踪你的双手，
快速挥手产生"手刀"，切开从屏幕底部喷出的西瓜得分，切到炸弹掉血，
3 条命用完游戏结束。连击有额外加分，金西瓜 +50。

启动：双击 `切水果.command`（macOS）/ `切水果.bat`（Windows），或命令行：

```bash
python -m fruitgame            # 默认摄像头
python -m fruitgame --camera 1 # 指定摄像头
python -m fruitgame --mouse    # 鼠标调试模式（不开摄像头）
```

- 全程不需要键盘：菜单和结算界面也是挥手切西瓜来操作
- 按键备选：空格开始、P 暂停、F 全屏、R 再来一局、Esc 退出
- 站远一点（1~2 米）、让上半身入画，识别最稳；挥动要快，慢慢划过是切不动的
- 最高分记在 `fruitgame/highscore.json`
- 玩游戏前建议先暂停坐姿监控（菜单栏点"暂停"），避免两边抢摄像头

## 星核指挥家：全身体感节奏游戏

站到摄像头前，用身体完成约两分钟的星际乐曲。青色目标对应屏幕左侧的手，
橙色目标对应屏幕右侧的手；紫色星桥需要双手同步，金色星门需要下蹲，
星座轮廓需要展开双臂定格。连续命中会逐层加入鼓点、旋律和琶音，
星核能量充满后进入 8 秒超新星状态。

### 启动

双击 `星核指挥家.command`（macOS）/ `星核指挥家.bat`（Windows），或在已激活
虚拟环境的项目目录中运行：

```bash
python -m stargame                    # 默认摄像头
python -m stargame --camera 1         # 指定摄像头
python -m stargame --reduced-motion   # 减少粒子和脉冲效果
python -m stargame --debug            # 键盘调试，不打开摄像头
```

### 使用步骤

1. 站在摄像头前约 1.5～2.5 米处，确保双手、肩部和髋部完整入画。
2. 在菜单中把双手举过头顶并保持 1 秒；也可以按空格键开始。
3. 按画面提示自然站立约 2 秒完成身体校准，校准期间双手放松，不要下蹲或倾斜。
4. 三秒倒计时结束后，观察从远处接近的星符，在它抵达目标圆环时完成对应动作。
5. 乐曲结束后会显示等级、得分和最高连击；再次举起双手 1 秒即可重玩。

### 体感操作与规则

| 画面目标 | 操作方法 | 判定重点 |
|---------|---------|---------|
| 青色流星 | 用屏幕左侧的手触碰目标圆环 | 位置和节拍时机 |
| 橙色流星 | 用屏幕右侧的手触碰目标圆环 | 位置和节拍时机 |
| 紫色星桥 | 双手同时进入左右两个目标 | 双手同步 |
| 金色星门 | 下蹲，让肩部明显降低 | 下蹲幅度和时机 |
| 紫色星座 | 双臂在肩部附近水平展开 | 手臂跨度和稳定性 |

- 判定分为 `PERFECT`、`GREAT`、`GOOD` 和 `MISS`；连续获得前两种判定会增加连击。
- 命中会补充星核能量，能量充满后进入 8 秒超新星状态，获得更高分数倍率。
- 星核能量归零后需要连续命中 3 次恢复；第三次失速会提前结束本局。
- 按 `P` 暂停或继续、`F` 切换全屏、`Esc` 返回菜单。
- `--debug` 模式使用 A/D 模拟左右手、J 模拟双手、S 模拟下蹲、W 模拟星座。

## 霓虹疾跑：三轨体感跑酷

采用经典三轨跑酷规则：角色自动前进，玩家倾斜身体切换轨道，双手上举跳过路障，
下蹲滑过横杆，快速挥手击碎木箱。收集金币并拾取护盾、磁铁、极速滑板和双倍积分，
速度会在三分钟内逐渐提升；三个生命耗尽后结算距离、金币与最高连续闪避。

### 启动

双击 `霓虹疾跑.command`（macOS）/ `霓虹疾跑.bat`（Windows），或在已激活
虚拟环境的项目目录中运行：

```bash
python -m runnergame                    # 默认摄像头
python -m runnergame --camera 1         # 指定摄像头
python -m runnergame --reduced-motion   # 减少扫描线、粒子和跑步摆动
python -m runnergame --debug            # 键盘调试，不打开摄像头
```

### 使用步骤

1. 站在摄像头前约 1.5～2.5 米处，让双手、肩部和髋部完整入画。
2. 在菜单中把双手举过头顶并保持 1 秒；也可以按空格键开始。
3. 自然站立约 2 秒完成校准。校准会记录身体中心、肩宽和正常站立高度。
4. 倒计时结束后角色会自动奔跑，根据前方障碍提示提前换道或完成对应动作。
5. 三颗生命全部耗尽后结算距离、金币、得分和最高连续闪避；举手 1 秒即可重玩。

### 体感操作

| 玩家动作 | 游戏行为 | 使用场景 |
|---------|---------|---------|
| 上身向左或向右倾斜 | 向对应方向切换一条轨道 | 躲避列车和普通障碍 |
| 双手快速举过头顶 | 跳跃约 0.9 秒 | 越过橙色路障 |
| 下蹲并保持 | 滑行 | 穿过紫色横杆 |
| 快速挥动任意一只手 | 向前挥击 | 击碎木箱 |

倾身换道后建议先回到自然站姿，再进行下一次换道；跳跃动作完成后也要先放下双手，
再次举手才能触发下一次跳跃。

### 规则与道具

- 双障碍波次始终至少保留一条安全轨道，金币排列会引导安全方向。
- 成功躲避或击碎障碍会增加连续闪避和分数；碰撞会中断连击并损失一颗生命。
- 护盾可以抵挡一次碰撞，最多同时持有两个。
- 金币磁铁持续 8 秒；极速滑板持续 6.5 秒并提供无敌冲刺；双倍积分持续 9 秒。
- 游戏速度会随时间逐渐增加，连续闪避和冲刺时音乐也会加入更多声部。
- 按 `P` 暂停或继续、`F` 切换全屏、`Esc` 返回菜单。
- `--debug` 模式使用 A/D 换道、空格跳跃、按住 S 下蹲、K 挥击。

### 常见问题

- **摄像头打不开**：先暂停或退出坐姿监控和其他体感游戏，避免多个程序同时占用摄像头；
  外接摄像头可尝试 `--camera 1`。
- **一直停在校准界面**：增加环境光线，后退一些，并确保双手、肩部和髋部没有出画。
- **动作频繁误触**：按 `Esc` 返回菜单后重新开始校准；校准时保持自然站立。
- **动态效果令人不适**：启动时增加 `--reduced-motion`，减少粒子、扫描线和背景脉冲。
- 两款游戏的摄像头画面只在内存中处理，不保存或上传。

## 代码结构

- `config.py` — 配置默认值和读写
- `posture.py` — MediaPipe 姿态检测 + 角度计算
- `monitor_core.py` — 采样/判定/提醒状态机（CLI 和菜单栏共用）
- `alerts.py` — 弹窗、声音、通知横幅（按平台分支：macOS 用 osascript/afplay，
  Windows 用 MessageBox/winsound）
- `main.py` — CLI 入口（校准/预览/监控），跨平台
- `menubar.py` — macOS 菜单栏常驻应用（rumps）
- `tray_win.py` — Windows 系统托盘常驻应用（pystray）
- `fruitgame/` — 体感切水果游戏（`tracker.py` 手部追踪 / `game.py` 主循环 /
  `sprites.py` 程序化美术 / `sfx.py` 合成音效）
- `stargame/` — 星核指挥家（`notes.py` 节奏谱面 / `tracker.py` 全身追踪 /
  `game.py` 动作判定与渲染 / `sfx.py` 自适应音乐）
- `runnergame/` — 霓虹疾跑（`world.py` 三轨规则与波次 / `tracker.py` 全身追踪 /
  `game.py` 跑酷循环与渲染 / `sfx.py` 自适应音乐）

## 性能与隐私

- 每 10 秒只做一次姿态推理（CPU 单帧约几十毫秒），平均 CPU 占用 <1%
- 摄像头保持开启（指示灯常亮）但不持续处理画面；这是为了取帧即时，
  功耗远低于视频会议
- 所有画面只在内存中分析后立即丢弃，不保存、不上传；除首次下载模型外零联网
